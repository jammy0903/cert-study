# 정보처리기사 실기 3단원 - DB 암기 프레임

관계형 데이터베이스 파트를 빠르게 분류하고 암기하기 위한 1단원 압축본.

## 핵심 프레임

- DB가 나오면 먼저 4덩어리로 자른다: 테이블 자체 / 그림(ERD) / 연산 / 트랜잭션
- 문제를 보기 전에 먼저 어느 덩어리 문제인지 분류하면 암기와 회상이 훨씬 쉬워진다.

## 1. 테이블 자체

- 릴레이션 = 테이블
- 튜플 = 행
- 속성 = 열
- 도메인 = 그 열이 가질 수 있는 값의 범위
- 카디널리티 = 행 개수
- 차수 = 열 개수
- 스키마 = 구조, 설계도
- 인스턴스 = 실제 저장된 현재 데이터 상태
- 식별자 = 행 하나를 구별하는 키

### 이 파트는 이렇게 외운다

- 행은 튜플
- 열은 속성
- 값 범위는 도메인
- 행 개수는 카디널리티
- 열 개수는 차수
- 설계도는 스키마
- 실데이터는 인스턴스

### 핵심 한 줄

- 행 / 열 / 범위 / 개수 / 설계도 / 실데이터

## 2. 그림(ERD)

- 개체(Entity) = 관리 대상 그 자체
- 관계(Relationship) = 개체와 개체 사이 연결
- 속성(Attribute) = 개체나 관계의 특징
- 개체집합 = 같은 종류 개체들의 모음
- 관계집합 속성 = 관계 자체에 붙는 속성
- 링크/연결선 = 개체와 관계를 이어주는 선
- ER 모델 표기법(피터첸 표기법) = ERD 기호 체계

### 이 파트는 이렇게 외운다

- 사물 = 개체
- 사이 = 관계
- 특징 = 속성
- 모아놓은 것 = 개체집합
- 관계에 붙은 특징 = 관계집합 속성

### 핵심 한 줄

- ERD는 사물 / 사이 / 특징 그림

## 3. 연산

- 관계대수 = 절차적
- 관계해석 = 비절차적

### 기준

- 절차적 = 어떻게 할지까지 보인다
- 비절차적 = 무엇을 원하는지만 쓴다

### 관계대수 연산

- 선택(σ) = 행 뽑기
- 프로젝션(π) = 열 뽑기
- 조인(⋈) = 공통 속성으로 붙이기
- 카티션프로덕트(×) = 모든 조합
- 합집합(∪) = 합치기
- 차집합(-) = 빼기
- 디비전(÷) = 모두 만족하는 것만 남기기

### 핵심 한 줄

- 선택은 행 / 프로젝션은 열 / 조인은 붙이기 / 카티션프로덕트는 전부 곱

## 4. 트랜잭션

- 트랜잭션이 나오면 다시 3덩어리로 자른다: ACID / 회복 / 병행제어

### 4-1. ACID

- Atomicity = 전부 하거나 전혀 안 하거나
- Consistency = 전후 상태가 일관돼야 함
- Isolation = 서로 간섭하면 안 됨
- Durability = 끝난 결과는 남아야 함

### 4-2. 회복

- Rollback = 이전 상태로 되돌리기
- REDO = 커밋된 것 다시 반영
- UNDO = 잘못된 것 취소
- 즉각 갱신 = 바로 DB 반영
- 지연 갱신 = 로그에만 두다가 커밋 때 반영

### 4-3. 병행제어

- Locking = 잠그기
- 낙관적 검증 = 일단 하고 끝에서 검사
- 타임스탬프 순서 = 시간표 순서대로
- 다중버전 동시성 제어(MVCC) = 버전 여러 개 유지

### 핵심 한 줄

- 트랜잭션 = ACID / 회복 / 병행제어

## 최종 정리

- DB 구조냐
- ERD 그림이냐
- 연산이냐
- 트랜잭션이냐

이 4개로 먼저 분류한 뒤 세부 용어를 붙이면 훨씬 안 헷갈린다.
